1.2.5 CPU

When you run stress -c 2 on your tinyVM, you may see activity on only one CPU in the CPU graph. This happens because tinyVM may have only one physical CPU, or the second virtual CPU is not fully scheduled, so the load appears only on the active CPU.

ESXi %RDY Explanation

After connecting via SSH and running esxtop, the %RDY column shows the percentage of time a virtual CPU was ready to run but could not get scheduled on a physical CPU. A high %RDY indicates CPU contention on the host.

1.3.2 vswitch 10 minutes

  1. On the ESXi host client, go to Network > Virtual Switch.
  2. Select vSwitch0 and click Modify Parameters.
  3. Go to Traffic Shaping (Formation de trafic).
  4. Set the bandwidth limit to 100 Mbps and save.
  5. Run iperf between VMs to measure bandwidth.
  6. Return to Traffic Shaping and set bandwidth to 1 Gbps.
  7. Run iperf again to verify the new bandwidth.

1.4 VM Advanced

1.4.1 Migration

  1. Create a tinyVM named tinyVMmigXX on vSanDatastoreXX.
  2. Suspend the VM from the ESXi host client.
  3. Go to vSanDatastoreXX, locate the VM folder, and save the VM.
  4. Find the file tinyVM.vmx in the datastore; this is the in-memory data file.
  5. Check which other ESXi hosts share the same vSanDatastoreXX.
  6. Connect to another host sharing the datastore.
  7. Resume the VM from that host. The VM will load the in-memory file and continue execution.
  8. VM migration is now complete.

1.4.2 Snapshot

  1. Open the tinyVM console from the ESXi host client (IHM).
  2. Inside the VM, create a directory named TOTO.
  3. Create a snapshot of the VM and name it Snapshot1.
  4. In the VM console, create another directory named TUTU.
  5. Restore the VM to Snapshot1 to revert the state before TUTU was created.

    2.2.3 VM autostart 5 minutes

    The configuration values for VM automatic startup and shutdown is stored in the file /etc/vmware/hostd/vmAutoStart.xml. By default auto start is not enabled. Change it ! 

    
<ConfigRoot>
<AutoStartOrder>
<_length>0</_length>
<_type>vim.host.AutoStartManager.AutoPowerInfo[]</_type>
</AutoStartOrder>
<SystemDefaults>
<_type>vim.host.AutoStartManager.SystemDefaults</_type>
<enabled>true</enabled>
<startDelay>120</startDelay>
<stopAction>PowerOff</stopAction>
<stopDelay>120</stopDelay>
<waitForHeartbeat>false</waitForHeartbeat>
</SystemDefaults>
</ConfigRoot>

    Finish !
